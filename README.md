# 汽车评估决策树分析项目

## 项目概述

本项目基于汽车评估数据集（Car Evaluation Dataset），使用决策树分类算法和Bagging集成学习方法，对汽车的等级进行分类预测。项目对比了单个最优决策树与Bagging集成模型的性能差异，展示了集成学习在提升模型准确率和降低过拟合方面的优势。

## 数据集说明

- **数据文件**: `car_evaluation.csv`
- **样本数量**: 1728条记录
- **特征数量**: 6个特征

### 特征列表

| 特征名 | 含义 | 取值范围 |
|--------|------|----------|
| buying | 购买价格 | low, med, high, vhigh |
| maint | 维修价格 | low, med, high, vhigh |
| doors | 车门数量 | 2, 3, 4, 5more |
| persons | 载客人数 | 2, 4, more |
| lug_boot | 后备箱大小 | small, med, big |
| safety | 安全性 | low, med, high |

### 目标变量

- **class**: 汽车等级，分为 unacc(不可接受)、acc(可接受)、good(良好)、vgood(非常好) 四个类别

## 主要功能

### 1. 数据预处理

- 使用有序编码（OrdinalEncoder）对分类特征进行编码，保留业务含义的顺序关系
- 6个特征均按照业务逻辑定义正确的排序进行编码

### 2. 特征工程

新增3个组合特征以提升模型表现：

- **total_cost**: 总拥有成本 = 购买价 + 维修价
- **utility**: 综合实用性 = 载客数 + 后备箱大小
- **cost_performance**: 性价比 = 实用性 / (总拥有成本 + 1)

### 3. 模型训练

- **单个决策树**: 通过网格搜索（GridSearchCV）寻找最优参数组合
- **Bagging集成**: 使用50个深度为4的决策树作为基学习器进行集成

### 4. 模型评估

- 计算训练集和测试集的准确率
- 分析过拟合程度
- 输出详细分类报告和混淆矩阵

### 5. 可视化

- 生成单个最优决策树的可视化图像

## 运行方式

```bash
python test_tree.py
```

## 输出示例

运行程序后会输出：

1. 数据集基本信息（样本数、特征数、类别分布）
2. 单个决策树的最优参数和交叉验证准确率
3. 两种模型的性能对比（训练集/测试集准确率、过拟合程度）
4. Bagging模型的详细分类报告和混淆矩阵
5. 决策树可视化图像

## 关键结果指标

程序会输出以下对比指标：

- 训练集准确率
- 测试集准确率
- 过拟合程度（训练准确率 - 测试准确率）
- 5折交叉验证准确率

## 注意事项

- 代码中设置了随机种子（random_state=42）以保证结果可复现
- 决策树可视化使用了中文字体设置
- 分层抽样（stratify）保证训练集和测试集的类别分布一致
